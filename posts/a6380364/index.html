Maven 入门

介绍

Maven 是Java项目的自动化构建工具，依托于项目对象模型POM(即Project Object Model)，为项目的全生命周期管理提供全方位支持。

官网下载并安装

Download Apache Maven

Windows 选择 Binary zip archive 下载apache-maven-x.x.x-bin.zip

解压压缩包并安装到指定目录，比如 Windows 可以解压安装到 "D:\Program\apache-maven-x.x.x\"

P.S. 其中 x.x.x 是版本号，下同

配置环境变量

Windows 桌面右键 “此电脑”，选择 “属性”，之后点击 “高级系统设置”，选择 “环境变量”

① 先新建系统变量MAVEN_HOME

  • 变量名 MAVEN_HOME

  • 变量值 D:\Program\apache-maven-x.x.x

P.S. 从 Maven 3.5.0 起，M2_HOME已废弃使用

② 再编辑系统变量Path,添加

  • 变量值 %MAVEN_HOME%\bin

③ 检查 Maven 是否安装成功

Win+R 输入cmd回车 或 Win+R 输入powershell回车

安装目录结构

bin

存放 Maven 的可执行文件

  • mvn 和 mvn.cmd：用于执行Maven构建命令来完成编译、测试、打包、部署等操作的脚本
  • mvnDebug 和 mvnDebug.cmd：用于以调试模式启动 Maven 构建过程的脚本

boot

存放 Maven 启动时所需的类加载器相关的 JAR 文件

  • plexus-classworlds-x.x.x.jar：用于实现类加载隔离机制的一个 Java 类库

conf

存放 Maven 的配置文件

  • settings.xml：Maven 的全局配置文件。全局级配置在 Maven 安装目录的conf文件夹下，用户级配置在用户目录下的.m2文件夹里，用户级配置会覆盖全局级配置
  • toolchains.xml： 工具链配置文件，用于解决在项目构建过程中对不同版本工具（如 JDK、Ant 等）的使用需求
  • logging：日志配置文件，用于控制 Maven 构建过程中的日志输出级别和格式

lib

存放 Maven 运行时所依赖的各种 Java 类库（JAR 文件）

配置settings.xml

配置本地仓库 LocalRepository

本地仓库默认位置在：C:/Users/当前用户名/.m2/repository

先在 Maven 安装目录下新建repo文件夹,然后编辑settings.xml文件

配置阿里云 Maven 镜像

  • <id>：镜像仓库唯一标识符，用于区分不同的镜像配置
  • <name>：镜像仓库描述性名称
  • <url>：镜像仓库 URL 地址
  • <mirrorOf>：指定代理的远程仓库，* 表示替代所有仓库

请参考：阿里云 Maven 仓库

IntelliJ IDEA 配置 Maven

IntelliJ IDEA 每次打开一个项目后，文件==>设置==>构建、执行、部署==>构建工具==>Maven，选择好Maven主路径后勾选用户设置文件和本地仓库的重写还有使用.mvn/maven.config中的设置

每次创建并打开新的 Maven 项目后，项目都需要重新这样配置，其实有个方法，如下：
文件==>关闭项目==>自定义==>所有设置==>构建、执行、部署==>构建工具==>Maven，选择好Maven主路径后勾选用户设置文件和本地仓库的重写还有使用.mvn/maven.config中的设置

最后控制台输出系统属性和环境变量信息
Win+R 输入cmd回车 或 Win+R 输入powershell回车

IntelliJ IDEA 使用模板 archetype 创建 Maven 项目

Maven 项目模板 archetype 推荐：
① maven-archetype-quickstart
② maven-archetype-webapp

文件==>新建==>项目==>Maven Archetype
填写项目信息并选择合适模板后创建
